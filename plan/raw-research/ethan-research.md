# Ethan's Research

**Overall Design/UI**
- Vibrant design with clear theme (theme examples include: wild west, ocean, ancient egypt, casino/money, wilderness).
- Large button or lever with animation (provides feedback, more stimulative/enaging).
- Actual slots (spinning part) should take up the largest part of the screen/playing area.
- History of winnings may not be good (if users lose more than win, depends on how RNG is set).

**User Specific Mechanics**
- Gambling -> less features and should prioritize actual slots, skewed RNG but still random.
- For fun -> more likely to win, added features like "power-ups" and a shop, completely random RNG, previous.
- Both -> should be very responsive, no more than 5 seconds per spin, multipliers and vibrant design with sound effects.

**Ideas to add while Prompting**
- Shop/Power-ups: Added shop with power-ups that can be a bonus multipler (ex: power-up where if you win in next 3 spins, your winnings are multiplied by 2x).
- Spinning Slots: Slots will obviously spin, although to build anticipation could have them slowly tick towards end.
- Specific themes: As mentioned earlier an overall theme with messages, sound effects, and design related to that theme (ocean -> bubble or sea creature sound effects).
- Flashing Lights: Lights around the machine can flash during the spin or if you win, same with things like confetti or other stimulating features.

**Technical Aspects**
- Overall Structure: Should follow basic styling guidelines, we should be able to easily add on if we wanted to implement a feature ourselves, follows implemenation guideline we saw in class.
- JS, Testing, and Frameworks: Should likely not use additional frameworks we are not familair with. JavaScript should be very responsive and interactive (large lever or button when betting). Any features implemented should all have testing to make sure it works, balance should be tested to make sure you can't go under and if you have a large sum it should not break the CSS. Things like JavaScript being disabled in settings should also be accounted for and different viewports/screen sizes.
